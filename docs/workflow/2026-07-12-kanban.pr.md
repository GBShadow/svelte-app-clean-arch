# Implementação do Quadro Kanban Reativo e Seguro

Created: 2026-07-12


> Copie este conteúdo para o body do Pull Request no GitHub.

## Resumo

Implementação completa do quadro Kanban reativo integrado com o Planning Poker no app `runes`. A funcionalidade suporta arrastar e soltar (drag and drop) de cards e colunas intermediárias (com total acessibilidade), edição rica com editor Tiptap, múltiplos responsáveis vinculados de forma segura, comentários em texto puro, e histórico imutável para auditoria das modificações.

## Alterações

### Domínio
- `apps/runes/src/lib/domain/kanbanAccess.ts` — Lógica pura de permissões e algoritmo de reordenamento contíguo de posições.
- `apps/runes/src/lib/domain/KanbanBoard.svelte.ts` — Classe reativa Svelte 5 para sincronização realtime e dedup de eventos.

### App(s)
- `apps/runes/src/routes/kanban/+page.server.ts` — Load de dados seguro e 100% das mutações via Server Actions, com validação Zod e sanitização de HTML.
- `apps/runes/src/routes/kanban/+page.svelte` — Interface do Kanban integrada com `svelte-dnd-action`.
- `apps/runes/src/lib/components/kanban/RichTextEditor.svelte` — Editor Rich Text modularizado utilizando Tiptap.
- `apps/runes/src/lib/appRegistry.ts` — Registra o app Kanban no Hub de aplicações.
- `pocketbase/pb_migrations/0015_create_kanban_collections.js` — Migrations com as tabelas do Kanban e colunas iniciais ("Aguardando", "Fazendo", "Feito").

### Testes
- `apps/runes/src/lib/domain/kanbanAccess.test.ts` — Cobre regras de acesso e algoritmo de posições.
- `apps/runes/src/lib/domain/KanbanBoard.test.ts` — Cobre a sincronização de eventos realtime.
- `apps/runes/src/lib/validation/kanbanSchemas.test.ts` — Cobre validações Zod.
- `apps/runes/e2e/kanban.spec.ts` — Testes E2E com Playwright cobrindo criação, edição, comentários, exclusão de cards e gerenciamento de colunas (admin).
- `apps/runes/e2e/cleanup.ts` — Limpeza de cartões e colunas criadas no banco de testes.
